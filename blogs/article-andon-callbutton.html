Replacing 100m Wired Runs with a Wireless Microcontroller Andon System

At Toyota Automation's Bengaluru facility, 38 call-button stations were each hardwired over 100 metres of cable back to a central PLC server. Replacing that wired backbone with a microcontroller-based wireless mesh — using a microcontroller and nRF24L01 transceivers — cut installation cost, eliminated cable runs, and retained full Andon voice annunciation and team-leader escalation logic.

§01 — The Problem: 38 Stations × 100m of Wire

Production lines at Toyota's client sites use Andon call-button stations to let operators signal for Team Leader or Maintenance support. Each station had two illuminated push-buttons — red for Team Leader, yellow for Maintenance — and each was wired all the way back to a central PLC controller acting as the Andon server.

⚠️ The legacy constraint: With 38 stations spaced at minimum 100 m intervals, the wired topology required thousands of metres of industrial cable, complex conduit routing, and a full PLC I/O expansion rack just to handle button states. Any station move meant re-cabling.

Fig 1 — Legacy wired topology (38 × 100m cable runs to PLC) vs. wireless nRF24L01+ mesh. Animated teal rings = live RF broadcasts; red glowing buttons = active calls.

The system also had strict behavioural requirements: if a call went un-attended for 60 seconds, it had to automatically escalate — Team Leader calls would route to Group Leader, and Maintenance calls would escalate to the Maintenance Team Leader. Voice annunciation through six horn speakers cycled through all active calls until each was attended.

💡 Goal: Prove a wireless concept using a microcontroller prototype — validate toggle buttons, illuminated acknowledge, 60-second escalation, voice annunciation — without any PLC or wired backbone, and produce a manufacturable PCB.

§02 — System Architecture

The redesigned system broke into three physical tiers connected over a 2.4 GHz RF mesh:

Fig 2 — 3-tier architecture. Teal dots = data packets flowing station → relay → controller. Gold dots = ACK packets returning to toggle the station's button LED off.

Each call station node carried one microcontroller, one nRF24L01 2.4 GHz module, two illuminated push-buttons, and a 5V supply. A button press packaged a 6-byte RF payload — station ID, button type, press timestamp — and broadcast it to the central controller. The node listened for an ACK packet that toggled its button LED off on successful attendance.

The central controller maintained a call queue, drove the Andon display over serial, triggered the voice synthesiser, and ran the 60-second escalation timer per active call.

§04 — Firmware & Core Logic

Fig 4 — Live state machine simulation. States cycle every 6 seconds: Idle → Call Active → Escalating → back to Idle. Watch the LED indicator and timer bar animate in sync.

Button Debounce & Toggle State Machine

Each button ran a software debounce filter — 20ms stable window — before registering a press event. A local toggle flag tracked whether the call was active or attended:

// Simplified call-button toggle logic (MCU node) bool callActive = false; unsigned long callStart = 0; void loop() { if (buttonPressed()) { callActive = !callActive; digitalWrite(LED_PIN, callActive); if (callActive) { callStart = millis(); rfSend({STATION_ID, TYPE_TL, CALL_ON}); } else { rfSend({STATION_ID, TYPE_TL, CALL_OFF}); } } // Non-blocking: blink LED if un-attended > 60s if (callActive && (millis() - callStart > 60000UL)) blinkLED(); }

Escalation Logic (Central Controller)

The central controller maintained a per-station call registry with timestamps. Every 5 seconds it scanned all active calls. Any TL call older than 60 seconds triggered a voice-index swap — the synthesiser began announcing the Group Leader voice clip instead. Similarly, a Maintenance call escalated to the Maintenance Team Leader clip.

📌 Time-gating: The entire call-button feature was suppressed outside shift hours — no RF payloads were processed during Tea Break, Lunch Break, shift change windows, or non-shift timing. A real-time clock module on the central controller gated the active processing window.

Fig 4b — Voice annunciation round-robin across 6 horn speakers. Highlighted speaker = currently active clip. Animated rings show audio propagation. RTC clock (right) gates the entire system — RF payloads are silently dropped during Tea Break, Lunch, and off-shift windows.

§05 — PCB Design with KiCad

To move beyond a breadboard prototype, the node circuit was laid out as a proper 2-layer PCB using KiCad EDA. This involved drawing the complete schematic in EESchema, assigning footprints via CvPcb, routing all signal and power nets in PCBNew, and running Design Rules Check with zero errors before generating Gerber files for manufacture.

Fig 5 — KiCad 2-layer PCB with animated trace routing. Teal = SPI signal, red = power rail, gold dots = vias (layer change). GND copper pour fades in after all traces complete. DRC badge appears green on successful pass.

The working prototype convinced the Toyota engineering team that the PLC-centric backbone could be phased out for new deployments. The demonstration also opened a direct conversation about providing in-house embedded solutions alongside their existing PLC automation work — a first for the company at that time.

📋 Bill of Materials outcome: A full BOM was produced during the internship — the first time the team had a component-level breakdown for a wireless Andon node. Each component was spec-verified against the design schematic, with cost-optimised alternatives evaluated where original parts were either unavailable or over-specified for the application.

Fig 6 — BOM unit cost breakdown per wireless node (animated bar chart). Total ≈ ₹970 vs ₹8,000+ for a PLC I/O expansion card — 88% per-station saving. Right: breadboard prototype layout (Microcontroller + nRF24L01+ + push-buttons + critical 10µF decoupling cap).

§07 — Key Technical & Professional Learnings

  • Schematic → PCB workflow: Learned end-to-end EDA — from net label best practices in EESchema to copper pour strategy and Gerber verification in PCBNew.
  • RF system design: Discovered that nRF24L01 range is highly sensitive to power supply decoupling; adding a 10µF bulk capacitor close to the module VCC pin resolved intermittent packet loss on the breadboard prototype.
  • BOM discipline: Preparing a cost-effective BOM taught specification literacy — understanding datasheet parameters, identifying pin-compatible substitutes, and flagging long-lead components early.
  • Escalation state machines: Implementing time-based escalation on a resource-constrained MCU (no RTOS) required careful use of millis() non-blocking timers rather than delay(), so the main loop could continue servicing the RF interrupt.
  • Industrial context awareness: Factory-floor systems have non-functional requirements that textbooks omit — shift-hour gating, noise immunity, button debounce for gloved hands, and speaker placement for intelligibility across a noisy weld shop.
