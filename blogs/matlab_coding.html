§ 01

Basic MATLAB — The Engineer's Computing Environment

MATLAB (MATrix LABoratory) is a high-level programming language and interactive environment built for numerical computation, data analysis, algorithm development, and visualization. Unlike general-purpose languages like C or Python, MATLAB is designed from the ground up around matrix and array operations — which makes it uniquely powerful for engineering mathematics where data lives naturally in vectors, matrices, and multi-dimensional arrays.

MathWorks ships MATLAB with hundreds of domain-specific toolboxes — from Signal Processing and Control System to Deep Learning and Robotics — so engineers rarely need to build core algorithms from scratch. The same MATLAB code runs on laptops, servers, and can be deployed to embedded hardware or the cloud.

💡 Why engineers choose MATLAB: The MATLAB syntax is designed so that a matrix equation like A \ b solves the linear system Ax = b directly — no loops, no library imports. What takes 20 lines in C takes 1 line in MATLAB. That speed-of-thought advantage is why MathWorks has 5 million+ users across 180 countries.

MATLAB R2024b IDE — three core panels: Command Window (interactive REPL), Script Editor (saved .m scripts), and Workspace (live variable inspector). This FFT snippet creates a 50 Hz sine wave, computes its frequency spectrum in one line, and plots the result.

Core MATLAB Concepts

🔢
Matrices & Arrays
Everything in MATLAB is a matrix. Scalars are 1×1, vectors are 1×n, images are m×n×3. Operations like +, *, / work element-wise or matrix-wise with a single character difference.
📈
Built-in Plotting
plot(), surf(), contour(), imagesc() — publication-quality figures in 1–3 lines. No matplotlib configuration needed; axes, labels, and legends are first-class citizens.
📦
Toolboxes
Signal Processing, Control System, Image Processing, Statistics, Deep Learning, Simulink — purpose-built algorithms ready to use. No pip install, no dependency hell.
🔄
Live Scripts
MATLAB Live Editor blends code, output, equations (LaTeX), and rich text in one document — perfect for engineering reports and analysis notebooks.
⚡
Vectorization
Replace for-loops with vector operations for 10–100× speed gains. y = sin(x) computes sine for every element in one call — no loop needed.
🚀
Code Generation
MATLAB Coder converts .m scripts to C/C++ for embedded deployment. Simulink Embedded Coder generates production-quality firmware from block diagrams — no hand-coding.

Essential MATLAB Syntax

matlab
%% === MATLAB BASICS: Core syntax reference ===

% 1. Variable assignment (no type declaration needed)
x = 42;                    % semicolon suppresses output
name = 'Sandeep';           % strings use single quotes

% 2. Matrix creation
A = [1 2 3; 4 5 6];       % 2×3 matrix (semicolon = new row)
v = 0:0.1:10;              % row vector from 0 to 10, step 0.1
I = eye(4);                 % 4×4 identity matrix
Z = zeros(3,3);            % 3×3 zero matrix

% 3. Matrix operations
C = A * A';                 % matrix multiply A by its transpose
d = det(C);                 % determinant
x = A \ b;                  % solve linear system Ax = b
e = eig(C);                 % eigenvalues

% 4. Flow control
for i = 1:length(v)
    y(i) = sin(v(i)) * exp(-0.1*v(i));
end
% Vectorized equivalent (100× faster):
y = sin(v) .* exp(-0.1*v);

% 5. Plotting
figure; plot(v, y, 'b-', 'LineWidth', 2);
xlabel('Time (s)'); ylabel('Amplitude');
title('Damped Sine Wave'); grid on;
§ 02

Simulink — Model-Based Design & Simulation

Simulink is MathWorks' graphical modelling environment for dynamic systems. Instead of writing equations as code, you build them as block diagrams — drag-and-drop blocks that represent mathematical operations, signal sources, physical components, and logic. Simulink then simulates the time-domain behaviour of your model, from milliseconds to years, with configurable solvers.

Simulink is used across aerospace (Airbus, Boeing), automotive (Bosch, Continental, Tesla), medical devices, energy, and industrial automation. It is not a toy — it is the tool that generates flight-certified autopilot code and production ECU firmware for millions of vehicles.

⚠️ Simulink vs MATLAB: MATLAB is a programming language; Simulink is a graphical modelling tool. They work together — MATLAB workspace variables feed Simulink simulations, and Simulink outputs can be analysed in MATLAB. But they solve different problems: MATLAB for algorithms and data analysis; Simulink for time-domain dynamic system simulation.

Simulink closed-loop PID control block diagram: a Step input (setpoint r(t)) feeds a summing junction. The error signal drives a PID Controller block, whose output drives a Transfer Function block modelling a DC motor. The Scope captures output y(t). A Sensor/Feedback block closes the loop back to the summing junction. The animated dot traces signal flow in real time.

Simulink Workflow

1️⃣
Model
Build the Block Diagram
Drag blocks from the Simulink Library Browser — sources (Step, Sine, Signal Builder), maths (Gain, Integrator, Transfer Fcn), logic (Switch, Stateflow), I/O (To Workspace, Scope). Connect them with signal lines.
2️⃣
Configure
Set Solver & Simulation Parameters
Choose a continuous solver (ode45, ode23, ode15s for stiff systems) or fixed-step solver (for embedded code generation). Set simulation start/stop time and step size.
3️⃣
Simulate
Run & Analyse
Click Run. Scopes show time-domain plots live during simulation. Signal logging captures every wire's time history to the MATLAB workspace for post-processing with plot(), fft(), or controlSystemAnalyzer().
4️⃣
Generate
Auto-Generate Production Code
Simulink Coder converts the model to portable C. Embedded Coder adds hardware-specific optimisations (ARM Cortex, TI C2000, STM32) and generates AUTOSAR or MISRA-C compliant code — ready for the ECU.

§ 03

Simulink for Electrical Engineering

Simscape Electrical (formerly SimPowerSystems) extends Simulink with physical modelling blocks for electrical circuits, power electronics, machines, and grid systems. Instead of writing KVL/KCL equations by hand, you build the circuit schematically — resistors, capacitors, inductors, diodes, MOSFETs, transformers — and Simscape solves the differential equations automatically.

This toolbox is used by power electronics engineers designing inverters, motor drives, PFC circuits, and battery management systems. The same model that verifies your converter topology can generate the embedded firmware for your DSP controller.

Simscape Electrical — Buck converter model: 48V input stepped down to 24V output via MOSFET switch, inductor, freewheeling diode, and output capacitor. The PWM Generator controls switch duty cycle at 100 kHz. Simscape solves the circuit equations numerically — ripple, efficiency, and transient response are measured automatically by sensor blocks.

What Electrical Engineers Simulate in Simulink

⚡
Power Electronics
Buck, boost, buck-boost, flyback, full-bridge, LLC resonant converters. MOSFET/IGBT switching losses, EMI prediction, efficiency sweeps across load range.
🔋
Battery & BMS
Equivalent circuit models (Thevenin, ECM), State of Charge/Health estimation, thermal models, cell balancing algorithms — the core of EV battery management.
🔌
Motor Drives
BLDC, PMSM, induction motor drives. Field-Oriented Control (FOC), SVPWM generation, Clarke/Park transforms — all blockset-ready.
🌞
Renewable Energy
Solar PV I-V curve models, MPPT algorithms (P&O, INC), grid-tied inverter control, wind turbine generators, microgrid management.
🏭
Smart Grid
Three-phase AC power flow, harmonic analysis, fault simulation (3-phase short, single-line-to-ground), protection relay coordination.
📡
RF & Mixed-Signal
LNA, mixer, PLL models in RF Blockset. ADC/DAC quantisation noise, sigma-delta modulator design, mixed-signal verification.
§ 04

Simulink for Electronics & Embedded Engineering

For embedded engineers, Simulink closes the gap between algorithm design and hardware deployment. You model your firmware logic as a block diagram, test it in simulation (software-in-the-loop), then push a button and get production-quality, MISRA-C compliant C code targeted to your specific MCU — STM32, TI C2000, NXP S32K, or custom hardware.

Stateflow, MathWorks' state machine environment embedded inside Simulink, models reactive control logic (mode transitions, fault handling, operating state machines) that would take hundreds of lines of hand-coded C. The same Stateflow chart simulates in Simulink and generates C code verbatim.

Model-Based Development pipeline: the algorithm starts as a Simulink model (MIL), becomes generated C code tested on PC (SIL), runs on the actual target MCU via JTAG (PIL), then integrates with a Hardware-in-the-Loop rig that simulates the physical plant (HIL). Each stage adds confidence before the next — and every test run generates certification evidence for DO-178C or ISO 26262.

Embedded Engineering Capabilities

⚙️
Stateflow
Graphical state machine and flow chart designer. Models operating modes, fault state machines, protocol parsers. Generates MISRA-C code directly — used in AUTOSAR components and safety-critical firmware.
💻
Embedded Coder
Generates optimised, portable C/C++ from Simulink models. Target-specific optimisations for ARM, TI C2000, NXP, Infineon. Produces AUTOSAR-compatible software components.
🧪
SIL / PIL Testing
Software-in-Loop runs generated C on PC for unit testing. Processor-in-Loop runs code on the actual MCU and compares outputs with the model to detect floating-point differences.
🔌
Hardware Support
Arduino, Raspberry Pi, STM32, TI LaunchPad, NXP FRDM support packages. One-click deploy to hardware for rapid prototyping and algorithm validation on real silicon.
📋
AUTOSAR Blockset
Design AUTOSAR Classic and Adaptive software components, runnable entities, and port interfaces directly in Simulink — integrated with arxml import/export for OEM toolchains.
🛡️
Polyspace
Static code analysis for run-time errors: overflow, divide-by-zero, out-of-bounds. Used for ISO 26262 and DO-178C software qualification — proves absence of run-time errors.
§ 05

Control Systems Using Simulink

Control system design is where MATLAB and Simulink shine brightest. The Control System Toolbox gives you transfer functions, state-space models, Bode plots, Nyquist diagrams, root locus, and automated PID tuning — all with 1–2 lines of MATLAB. Simulink turns those designs into closed-loop simulations you can test before touching hardware.

From a simple PID thermostat to a multi-loop cascaded flight control system, the workflow is the same: model the plant, design the controller, simulate the closed loop, tune, verify stability margins, and generate embedded code. The Control System Tuner app and PID Tuner automate the tuning step — finding optimal gains in seconds that would take hours of manual trial-and-error.

💡 Real project example: In my SLI crane project, I used MATLAB's Control System Toolbox to design and verify a load rate-of-change filter that suppressed dynamic load spikes without introducing measurement lag. The filter was designed in MATLAB (transfer function, Bode plot, phase margin verification), simulated in Simulink, then the generated C code was dropped into the STM32 firmware. Zero manual rewriting.

MATLAB Control System Toolbox analysis tools: Bode plot (left) shows gain and phase margins — GM=12dB means the loop can gain an extra 12dB before instability; Step response (centre) shows 20% overshoot and rise time from the PID-tuned closed loop; Root locus (right) shows how closed-loop poles move as gain increases, with the PID Tuner's auto-calculated gains achieving 60° phase margin.

Control System Design Workflow in MATLAB + Simulink

📐
Step 01
Model the Plant
Create a transfer function or state-space model of the system being controlled. In MATLAB: G = tf([K], [tau 1]) defines a first-order plant. Use system identification (ident toolbox) to extract models from real measured data.
📊
Step 02
Analyse Open-Loop Behaviour
Plot the Bode diagram (bode(G)), Nyquist diagram, step response, and pole-zero map. Understand gain crossover frequency, phase margin, and natural frequency before any controller is added.
🎯
Step 03
Design the Controller
Use pidTuner(G, 'PID') for automatic PID tuning, rlocus(G) for root locus, or loopsyn(G, W) for H∞ robust control. The PID Tuner app adjusts bandwidth/robustness sliders and shows the result live.
🔬
Step 04
Verify in Simulink Closed Loop
Import plant and controller blocks into Simulink. Add disturbances, noise, saturation limits, and anti-windup. Simulate step responses, disturbance rejection, and parameter variations. Check settling time, overshoot, and steady-state error against requirements.
🚀
Step 05
Deploy to Embedded Target
Use Embedded Coder to generate the discrete-time controller C code for your MCU. Choose discretisation method (Tustin, ZOH, matched poles), set the sample time, and generate code. The generated PID integrator handles anti-windup, bumpless transfer, and output limits automatically.

Types of Control Systems You Can Design

🔄
PID Control
The workhorse of industrial control. Auto-tune with pidTuner(), implement anti-windup, derivative filtering, and set-point weighting. Used in 90%+ of real embedded controllers.
🏗️
State-Space / LQR
Model multi-input multi-output (MIMO) systems as state-space equations. Design optimal controllers with LQR (linear quadratic regulator) and Kalman filter state estimators.
🔋
Model Predictive Control
MPC Toolbox designs controllers that optimise a cost function over a receding horizon — handles constraints explicitly. Used in battery management, HVAC, and process control.
🦾
Robust Control (H∞)
Design controllers that maintain stability and performance despite plant uncertainty. Used where the system model is imprecise or parameter variations are large.
🤖
Adaptive Control
Controllers that update their own gains based on system performance — useful when plant parameters change over time (e.g., aircraft changing mass as fuel burns).
📡
Digital Control
Discretise continuous controllers for DSP/MCU implementation. Tustin (bilinear) and matched-pole discretisation with configurable sample rates. Z-domain stability verification.
